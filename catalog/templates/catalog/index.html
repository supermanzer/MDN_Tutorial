This is a very basic Django page, built based upon the Mozilla Developer Network tutorial.

I find I continue to revise my ideas for how best to implement Django solutions by re-working simple tutorials. This provides me a simple app in which to try out new implementation ideas where the logic of all the interconnected parts is minimal.

The little tidbit about passing additioal options in the URL mappings to route multiple URLs to the same view with slighlty different behavior in Section 6 is a real gem and I'm sure I'll have a use for it soon.

Another idea worth a little testing is changing the session in the Django server side code and having a JavaScript listener detect the change and respond. While this wouldn't be as robust as using Django Channels, it would allow the updating of small UI components like the Upload Status of a file.
